Open Door

3D Puzzle

Role: Solo Developer Engine: Unreal Engine 5 Platform: PC
View on GitHub

Overview

Open Door is my first Unreal Engine project — a 3D puzzle game where the player collects three crystals scattered across the environment to unlock a door and win. Built to learn Unreal's gameplay framework, I implemented the core mechanics using both Blueprints for rapid iteration and C++ for the underlying game logic. This project was a deliberate step toward building expertise in Unreal alongside my Unity background.

Open Door - Trailer

Key Programming Features

  • → Crystal pickup system implemented in C++ with Blueprint-exposed properties for designer tweaking
  • → Door unlock logic triggered by a C++ gameplay event once all three crystals are collected
  • → Blueprint-driven level interactions — pressure plates, triggers, and animated door sequences
  • → First-person player controller using Unreal's Character class and Enhanced Input system
  • → Environmental puzzle design with spatial cues guiding the player toward crystals
  • → Basic UI displaying crystal collection progress using Unreal's UMG widget system

What I Learned

Open Door was my introduction to Unreal Engine's programming model and how it differs from Unity. Understanding the relationship between C++ base classes and Blueprint subclasses was the biggest mental shift. I intentionally kept the scope small so I could focus on understanding the framework — and it paid off. This is the foundation I'm now building on as I go deeper into Unreal.
