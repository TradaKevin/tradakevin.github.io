AR Marker

Augmented Reality

Role: Solo Developer Engine: Unity AR Foundation Platform: Android / iOS
View on GitHub

Overview

AR Marker is a marker-based augmented reality application where the device camera recognizes a specific reference image and projects an interactive 3D model on top of it in real time. When the marker is detected, the 3D model appears anchored to the physical image and responds to touch input. Built with Unity AR Foundation as a practical exploration of image tracking AR — one of the core techniques covered in my Mobile AR Development certification.

AR Marker- Trailer

Key Programming Features

  • → Image tracking using Unity AR Foundation's AR Tracked Image Manager to recognize the reference marker
  • → 3D model instantiation anchored precisely to the marker's real-world position and rotation
  • → Tracking state management — model shows/hides cleanly when the marker enters or leaves the camera view
  • → Touch input handling to trigger animations or interactions on the detected 3D model
  • → Reference image library setup in AR Foundation with configurable physical size for accurate scale
  • → Mobile build configuration for both Android and iOS with camera permission handling

What I Learned

AR Marker deepened my understanding of how AR tracking works under the hood — specifically how the framework maps real-world coordinates to Unity world space. Handling tracking loss gracefully (so the model doesn't just freeze mid-air when the marker is obscured) required writing proper state management, which was a valuable lesson in defensive programming for mobile apps.
